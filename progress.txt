## Codebase Patterns
- New training-regime pages follow the DPO slice: registry record under `src/content/registry/training-regimes/`, citation + graph records, page bundle under `src/content/docs/training/<slug>/`, and a focused `src/lib/content/<slug>-training-regime.test.ts` with live search routing checks.
- Run `bun run prepare:content-runtime` (or any `pretest`/`pretypecheck` hook) after adding registry records so `published-docs-registry.generated.ts` and `registry-runtime.generated.ts` pick up the new page.
- Training-regime `how-it-works` sections must include both `TrainingRegimeFlow` and `BlockMath` per `validate-registry.ts` template contract.
- Bidirectional alignment navigation: add `training-regime.grpo` to `concept.alignment`, `training-regime.dpo`, `training-regime.post-training`, and `classification.training.alignment` relatedIds when shipping a new alignment method page.
- Add new alignment training regimes to `recordsRequiringOntologyPrimaryClassification` in `src/lib/content/registry.ts` and extend `ontology-foundation-regression.test.ts` primary membership expectations.
- Isolation-first training-regime copy should define the method and post-training grouped-comparison problem in `whatItIs`/`whyItExists` before leaning on adjacent-method comparisons; add a dedicated render test asserting opening summary, first sections, and absence of page-meta prose.
- Story 003 grouped-loop teaching: expand `howItWorks` with prompt-to-samples-to-relative-update prose, add `math.groupedRelativeAdvantage` symbol definitions under `BlockMath` in page MDX (pretraining pattern), refine graph node labels, and mirror `post-training-training-regime.test.ts` graph/render contracts.
- Story 005 slice verification: add `grpo-training-regime-slice-verification.test.ts` mirroring `instruction-tuning-training-regime-slice-verification.test.ts` for route/registry bundle resolution, graph/citation resolution, live search, tag landing, and consolidated rendered surface proof; routine bundle invariants stay on `validateDerivedPublishedPageBundles`.
- GRPO canonical expansion per DeepSeekMath source is **Group Relative Policy Optimization**; keep `group relative preference optimization` as a discovery alias only.
- Unresolved search `classification` handoff params should not be passed to `useModelAtlasDocsSearch`; classification handoff result waits should use the same 15s Orama timeout helper as Phase 1 search tests.
