## Codebase Patterns
- Use `getDocsPageDir(section, slug)` for page tests; do not add new `*_PAGE_DIR` exports.
- Run `make validate-data` before registry-focused tests so `registry-runtime.generated.ts` exists.
- Module registry slices follow `multi-token-prediction-module-registry.test.ts`: schema parse, citation proof, curated related-docs with `PUBLISHED_DOCS_REGISTRY_IDS`.
- Place diffusion-related modules under `classification.module.transformer-block` with `moduleFamily: "transformer-block"` when describing denoising transformer blocks.
- Reuse existing citations (`citation.denoising-diffusion-probabilistic-models`) when factual claims already have published sources; add new citation/paper records only when missing.
- Paper records can ship in registry before page bundles (e.g. `paper.diffusion-transformers`); related-doc tests should only assert hrefs for targets in `PUBLISHED_DOCS_REGISTRY_IDS`.
