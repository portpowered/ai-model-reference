## Codebase Patterns
- Blog posts live under `src/content/blog/<slug>/` with `page.mdx`, `messages/en.json`, and optional `assets.json`; slug becomes `/blog/<slug>`.
- Reuse landed blog shell components: `T`, `TagPillList`, `BlogRelatedDocs`; import them in MDX frontmatter imports block.
- Embed `RooflineThroughputExplorerFromRegistry` for registry-backed roofline charts; presets resolve at build time via `getRooflineModelSizePresets()`.
- Published blog `relatedDocIds` and `tags` must resolve via registry validation (`make validate-data` / `bun run validate-data`).
- Colocated blog messages require `title`, `description`, and optional `contextSentence` / `takeaway` per `blogPostMessagesSchema`.
- When adding a published blog post, update `blog-content-loader-scope.test.ts` and `blog-tag-landing.test.ts` slug inventories (newest-first on tag pages).
- Blog inline canonical links use markdown `[label](/docs/...)` syntax; `validate-data` / `validatePublishedBlogPosts` rejects broken reader-visible MDX links.
- Blog search discovery tests live in `build-blog-search-document.test.ts` under a post-specific `describe` block; assert document fields, representative `docsSearchApi.search` queries, and tag-filtered search without full-site route inventories.
- Responsive blog post verification: RTL tests in `roofline-max-throughput-responsive-verification.test.tsx`; Playwright viewport probes in `src/lib/verify/roofline-max-throughput-blog-viewport-http.test.tsx` (main `make test` excludes `src/lib/verify/`—probes run under `test-verify-contract`).
