## Codebase Patterns
- Model pages live under `src/content/docs/models/{slug}/` with `page.mdx`, `messages/en.json`, and `assets.json`; follow `gpt-3` as the closest GPT-family decoder-only reference.
- Model registry records live at `src/content/registry/models/{slug}.json` with `id: model.{slug}`, family `gpt`, and `citationIds` pointing to existing citation records.
- Reuse existing `paper.gpt-2-report` and `citation.gpt-2-report` for GPT-2 source attribution; do not create duplicate paper/citation records.
- Run `bun run typecheck` for typecheck gate; `make validate-data` for registry/content validation after adding records.
- Page-local scope: avoid shared helper edits, generated artifact commits, and page-specific tests unless special behavior requires them.
- Model `sourceType` enum is `open-weights` | `closed` | `research` (not `public`); use `open-weights` for GPT-2 since smaller checkpoints were publicly released.
- Registry-only slices need no page bundle: `make validate-data` regenerates ignored runtime artifacts locally but they stay untracked.
