## Codebase Patterns
- Before authoring Whisper content, verify `src/content/docs/models/whisper/` and `src/content/registry/models/whisper.json` are absent or update only missing pieces; guard with `src/lib/content/whisper-duplicate-guard.test.ts`.
- Use `getDocsPageDir("models", slug)` and `getModelsDocsRoot()` for page-local path resolution instead of new exported constants.
- Run `bun run prepare:content-runtime` before focused content tests that import `registry-runtime`.
- Page-local workflow notes: `docs/internal/processes/whisper-model-page-relevant-files.md`.
- Model registry stories follow `src/lib/content/llama-3-model-registry.test.ts`: parse JSON with `modelRecordSchema`, assert runtime resolution via `getModelById`/`getCitationById`, and verify required `relatedIds` without editing unrelated training-regime reverse links unless validation requires it.
- Model page stories mirror `src/lib/content/llama-3-model-page.test.tsx`: validate bundle with `validateGeneratedPageBundle`, assert search ranking, curated related docs, and static render of sections/citations; add `ModelArchitectureGraph` assertions once story 005 wires `graph.whisper-architecture` and `assets.json`.
- Architecture graph stories mirror `src/lib/content/ltx-23-architecture-graph.test.tsx`: registry graph record, `buildRegistryFlowGraph` labels, static/hydrated `RegistryGraphFlow`, and model-page architecture section graph surface.
- Story 006 slice verification mirrors `flux-model-slice-verification.test.tsx` / `gemma-slice-verification.test.tsx`: consolidated proof for validateGeneratedPageBundle, citation/link resolution, search/tag discovery, and optional Playwright route verification in `src/lib/content/whisper-slice-verification.test.tsx`.
