## Codebase Patterns
- Registry Zod schemas live in `src/lib/content/schemas.ts`; shared base fields use a `baseRecordShape` object spread into kind-specific schemas with `z.literal("module"|"concept"|"tag"|"citation")` for `kind`.
- `registryRecordSchema` is a `z.discriminatedUnion("kind", [moduleRecordSchema, conceptRecordSchema, tagRecordSchema, citationRecordSchema])` for parsing any Phase 1 registry record by `kind`.
- Page schemas use `pageKindSchema` (docs page kinds) separate from `registryKindSchema`; `pageAssetSchema` is a `z.discriminatedUnion("type", [...])` and `pageAssetConfigSchema` is `z.record(z.string(), pageAssetSchema)`.
- `loadRegistry()` in `src/lib/content/registry.ts` reads `src/content/registry/{modules,concepts,tags,citations}/*.json`, validates per-directory Zod schemas, and returns `byId`, `bySlug`, `tagsById`, and `tagsBySlug` maps; pass `registryRoot` in tests for temp fixtures (always mkdir `concepts` in temp registry fixtures).
- `RegistryLoadError` carries structured `details` for duplicate id/slug (`duplicate-id`, `duplicate-slug`) and parse/validation failures (`parse-error`).
- `baseline-records.test.ts` reads committed JSON under `src/content/registry/` and validates with kind-specific Zod schemas via `readRegistryJson` helper; use for story-level proof that on-disk records match schemas.
- Phase 1 toolchain: Bun for install/test, `bun run typecheck` (`tsc --noEmit`), Biome for lint/format; colocate unit tests as `*.test.ts` next to the module under test.
- `loadPageMessages(pageDirectory, locale)` in `src/lib/content/messages.ts` reads `messages/<locale>.json`, validates with `pageMessagesSchema`, and throws `MessageLoadError` with `missing-file` or `parse-error` details; missing `en` uses an explicit default-locale error message.
- `loadPageAssets(pageDirectory)` and `resolvePageAsset(pageDirectory, assetId)` in `src/lib/content/assets.ts` read colocated `assets.json`, validate with `pageAssetConfigSchema`, and throw `AssetLoadError` with `missing-file`, `parse-error`, or `unknown-asset-id` details.
- `resolvePageAssetWithMessages(pageDirectory, assetId, locale?)` loads the asset plus default-locale messages and throws `AssetLoadError` with `missing-message-key` when `altKey`/`captionKey` do not resolve; use `getMessageString(messages, keyPath)` from `messages.ts` for dot-path lookups (shared with `validate-registry.ts`).
- Export `assetMessageKeys(asset)` from `assets.ts` when listing message keys on a `PageAsset` for validation or tests.
- `prd.json`, `prd.md`, and `progress.txt` are local workflow tracking only — do not commit them to the PR branch.
- Phase 1 starter registry set: `module.grouped-query-attention`, `concept.token`, `tag.attention`, `tag.kv-cache` (tag resolution for module discovery), `citation.gqa-paper`; module `tags` must include both `attention` and `kv-cache`.
- Phase 1 page directories: module at `src/content/docs/modules/grouped-query-attention/`, glossary at `src/content/docs/glossary/token/`; export `tokenGlossaryPageDir` alongside `groupedQueryAttentionPageDir` from `messages.ts`.
- Use `baseline-page-fixtures.test.ts` to assert both Phase 1 page dirs have template-aligned `messages/en.json` and message-backed `assets.json` keys.
- `validateRegistryContent()` in `validate-registry.ts` loads registry via `loadRegistry()` (unique id/slug), validates references for all registry kinds including `concepts/`, walks `page.mdx` under `docs/`, and always validates Phase 1 colocated bundles for directories in `phase1PageDirectories` that lack `page.mdx` (token glossary today).
- Export `validateColocatedPageBundle(pageDirectory)` for colocated-only checks; pass `phase1PageDirectories` in `ValidateRegistryContentOptions` to override in tests.
