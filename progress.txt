## Codebase Patterns
- Factory batch examples live at `factory/docs/batch-input-example.json` using canonical `FACTORY_REQUEST_BATCH` shape with top-level `requestId`, `type`, and `works[]`; use `workTypeName` (not `workType`) and camelCase relation fields `sourceWorkName` / `targetWorkName`.
- New registry records under `src/content/registry/` are picked up by `bun run prepare:content-runtime`, which regenerates `src/lib/content/generated/registry-runtime.generated.ts` before typecheck, lint, and tests run.
- Model registry records use typed relationship fields (`architectureIds`, `moduleIds`, `trainingRegimeIds`, `relatedIds`) plus backing `citation`, `paper`, and `organization` records; update reciprocal `usedByModelIds` / `relatedIds` on linked training regimes when adding model consumers.
- For post-training discovery when no generic `training-regime.post-training` record exists, link models to `concept.alignment` in `relatedIds` and keep `training-regime.pretraining` in `trainingRegimeIds`.
- Focused registry contract tests live beside other content tests in `src/lib/content/*-registry.test.ts` and should assert schema validity plus `getModelById` / `getPaperById` runtime resolution.
- New model pages under `src/content/docs/models/<slug>/` auto-register via `prepare:content-runtime`; use empty `{}` in `assets.json` until a graph asset is added in a later story.
- Model architecture graphs pair `src/content/registry/graphs/<slug>-architecture.json` with `assets.json` `architectureGraph.graphId` and localized `graph.nodes` / `assets.architectureGraph.alt` keys in `messages/en.json`; follow the DeepSeek-V4-Pro RoPE + RMSNorm graph pattern for modern decoder stacks.
- Qwen 3.6 model records use `draft` status until canonical model pages ship in later stories; published validation requires matching docs pages and release metadata.
- Shared Qwen primary sources should reuse one citation id (`citation.qwen36-github`) across both dense and MoE model records; model-specific blogs and Hugging Face cards stay separate citation ids.
- Registry `tags` must reference existing tag records under `src/content/registry/tags/`; there is no `mixture-of-experts` tag—use `feed-forward` for MoE-related discovery metadata until a dedicated tag exists.
- Official Qwen 3.6 names confirmed by primary sources: `Qwen3.6-27B` (dense, slug `qwen-3-6-27b`) and `Qwen3.6-35B-A3B` (MoE, slug `qwen-3-6-35b-a3b`); 262,144-token context and text/image/video inputs from Hugging Face model cards.
- Pair new model registry slices with focused runtime tests in `src/lib/content/` using `getModelById`, `resolveCitations`, and `listModelRecords` rather than meta file scans.
- Consolidate paired model-page slice verification in one `qwen-3-6-paired-slice-verification.test.tsx` using `loadModelPage`, `ModulePageProviders` static render, and `docsSearchApi` to prove routes, graph surface, shared citations, and dense-vs-MoE discovery together.
- Dense Qwen 3.6 architecture graphs should follow the DeepSeek RoPE + RMSNorm stack layout but swap MoE nodes for `module.standard-ffn` to teach dense activation without naming unexpanded hybrid blocks in graph labels.
- MoE Qwen 3.6 architecture graphs reuse the dense stack layout but replace the FFN sublayer with `moe-sublayer`, `rmsnorm-moe`, and an `moe-routing` node linked to `module.mixture-of-experts` for expert-routing emphasis.
- Model discovery paths should use registry `relatedIds` for curated links to prerequisite concepts, modules, and systems; pair dense/MoE counterparts in `relatedIds` first so `DerivedRelatedDocs` surfaces them under `curated-related`.
- Add the `attention` tag to model records when connecting them to the attention tag landing; MoE models can also use `feed-forward` for MoE module discovery.
- Critical docs autodiscovery render smoke walks every metadata-discovered page (currently 36+); keep `CRITICAL_DOCS_AUTODISCOVERY_RENDER_TIMEOUT_MS` at or above 45s when the attention or token-to-probability smoke sets grow.
- When merging `main` into a model-page branch, attention tag landing inventory tests may need both new models (e.g. llama-3 and Qwen 3.6) in the expected model-group list.
