## Codebase Patterns
- New registry citations need both a JSON record under `src/content/registry/citations/` and an entry in `src/lib/content/citations.ts` so `CitationList` can resolve MLA references at render time.
- `bun run generate:registry-runtime` (via pretest/pretypecheck) must run after adding registry JSON so `registry-runtime.generated.ts` stays current.
- Published model records require `releaseDate`, `authors`, `sourceId`, and at least one `citationIds` entry; use `getCitationById` from registry-runtime for citation lookups (citations are not in `getRegistryRecordById`).
- Connect model `relatedIds` and `architectureIds` only to registry ids that exist on the branch; CLIP currently ships as `citation.learning-transferable-visual-models-from-natural-language-supervision`, and latent-diffusion maps to `concept.latent` plus diffusion glossary concepts.
