## Codebase Patterns
- Registry Zod schemas live in `src/lib/content/schemas.ts`; shared base fields use a `baseRecordShape` object spread into kind-specific schemas with `z.literal("module"|"tag"|"citation")` for `kind`.
- Page schemas use `pageKindSchema` (docs page kinds) separate from `registryKindSchema`; `pageAssetSchema` is a `z.discriminatedUnion("type", [...])` and `pageAssetConfigSchema` is `z.record(z.string(), pageAssetSchema)`.
- Phase 1 toolchain: Bun for install/test, `bun run typecheck` (`tsc --noEmit`), Biome for lint/format; colocate unit tests as `*.test.ts` next to the module under test.
- `prd.json`, `prd.md`, and `progress.txt` are local workflow tracking only — do not commit them to the PR branch.
