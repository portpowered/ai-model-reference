## Codebase Patterns
- Cosmos 3 story 001 follows the Nemotron release-identity slice: `page-specs/cosmos-3.json`, primary-source citation records, `paper.cosmos-3`, and `cosmos-3-release-identity.test.ts` before the model registry record or MDX bundle land in later stories.
- Run `make validate-data` after adding registry citations/papers so `registry-runtime.generated.ts` exists before release-identity tests call `getCitationById` / `validateRegistryContent`.
- Concept registry ids route to `/docs/concepts/<slug>` only after adding to `PUBLISHED_CONCEPT_SECTION_REGISTRY_IDS` in `published-docs-registry-ids.ts`; until then glossary pages at `/docs/glossary/<slug>` remain the live surface.
- Serving-path glossary concepts (`prefill`, `decode`, `kv-cache`, `prefill-decode-split`) share tags `foundations`, `attention`, and `kv-cache` for attention-tag discovery without new taxonomy tags.
- Registry `relatedIds` on `concept.prefill-decode-split` should stay limited to serving foundations (`prefill`, `decode`, `kv-cache`, `autoregressive-generation`); attention modules and transformer links belong on inbound records, not on the split concept itself.
- Inbound serving-path glossary pages (`prefill`, `decode`) link to `/docs/concepts/prefill-decode-split`; update hardcoded `LocalizedLinkList` hrefs when migrating a glossary concept to the concepts tree.
- Concept page bundles require an `assets.json` file (even `{}`) because `loadConceptPageFromDisk` always reads it; run `bun run fumadocs-mdx` after adding or moving MDX so `.source` picks up the new route.
- Migrating a glossary page to concepts removes it from glossary index counts/tests and surfaces it under the attention tag `concept` kind group instead of `glossary`.
- Story 004 slice verification: add `*-slice-verification.test.tsx` with registry/message alignment, `renderDocsSlugPage` route proof, and rendered HTML/search assertions; routine bundle invariants stay in `make validate-data`; run `audit:canonical-page-surface` from the root repo with `--repo-root` pointing at the worktree when the script is absent locally.
- After merging `main`, regenerate published-docs manifests with `bun run generate:published-docs-registry` (or `make validate-data` / `prepare:content-runtime`) instead of hand-editing `published-docs-registry-ids.ts`.
