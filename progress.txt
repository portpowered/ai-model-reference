## Codebase Patterns
- Training-regime registry records live in `src/content/registry/training-regimes/{slug}.json` and auto-load via `loadRegistry`.
- Published training-regime records require `releaseDate`, `authors`, `sourceId`, and at least one `citationIds` entry.
- Reuse existing citation records (e.g. `citation.training-language-models-to-follow-instructions-with-human-feedback`) instead of duplicating sources.
- `regimeType: "post-training"` with `sidebarGrouping.training: "post-training"` groups post-pretraining regimes like specialist-training.
- Valid training-regime tags include `alignment` and `foundations` for post-training adaptation regimes.
- Serving metric glossary pages use `kind: glossary` with `concept.<slug>` registry records, `conceptType: inference`, and `sidebarGrouping.glossary: sequence-and-attention`; mirror `time-to-first-token` for aliases, relatedIds, and citations (`citation.orca-serving-system`, `citation.brown-gpt-3`).
- Published glossary discovery requires aligned registry record, page frontmatter, and `messages/en.json` title/description; `prepare:content-runtime` regenerates `published-docs-registry.generated.ts` on validate.
- Publishing `concept.inter-token-latency` with alias `latency` enables prose auto-links on other pages; brittle HTML substring tests should assert link targets or fragments that survive auto-linking.
- Serving metric comparison tables use `src/content/registry/tables/{slug}-metric-comparison.json` with `subjectId: concept.<slug>`, columns anchored to published registry ids (`concept.time-to-first-token`, `concept.inter-token-latency`, `system.batching` for tokens-per-second column), colocated `tables.comparison` messages, and `PageAsset` in `page.mdx`; `make validate-data` regenerates `table-registry.generated.ts`.
- Serving metric glossary related-routing pairs `relatedDocs` message overrides (layperson `reason` per `relatedIds` entry) with a `serving-path` `LocalizedLinkList` section; derived validation plus page-owned `audit:canonical-page-surface` is enough for routine metric pages unless special rendering needs a consolidated slice file.
- Metric comparison tables require committing `table-registry.generated.ts` after `make validate-data`; page-owned audit (`--files` on bundle + registry + table JSON only) should report `keep-routine` even when the full branch also carries mergeability-only shared-test touches.
- Alias `latency` on `concept.inter-token-latency` is the first concept-level prose auto-link for that phrase; update brittle downstream substring tests (for example `memory-system-page.test.ts`) to href/fragment assertions instead of adding slice verification churn.
