## Codebase Patterns
- Module docs pages use `loadModulePage(slug)` from `src/lib/content/module-page.ts`, which compiles MDX with `moduleMdxComponents` and loads colocated `messages/<locale>.json` plus `assets.json`.
- GQA reference route: `src/app/(site)/docs/modules/grouped-query-attention/page.tsx` wraps `loadModulePage` output in `ModulePageProviders` inside `(site)/layout.tsx` `DocsShell`.
- Wrap compiled MDX in `ModulePageProviders` so client components `T`, `Section`, `PageAsset`, and `Callout` receive messages/assets via React context.
- `PageMessagesProvider` defaults `isDev` from `process.env.NODE_ENV === "development"`; missing keys render `MissingMessageKey` in dev and nothing in production.
- Message lookup uses dot-path keys via `lookupMessage` / `resolveMessage` in `src/lib/content/messages.ts`.
- Asset resolution: `parsePageAssetConfig` + `lookupAsset` / `resolveAssetText` in `src/lib/content/assets.ts`; render with `PageAsset` (via `ModuleGraph`, `ModuleComparisonTable`, or directly). Invalid config throws at load; missing `assetId` shows `MissingAssetId` in dev.
- Run quality checks with `bun run typecheck`, `bun run lint`, and `bun test` after `bun install`.
- `TagPillList` resolves tags via `getRegistryTags(registryId)` or an explicit `tags` prop; pills link to `tagPageHref(slug)` (`/tags/<slug>`) with monochrome `secondary` + `border` styling.
- Citation records live in `src/content/registry/citations/*.json`; load via `getCitationById` / `resolveCitations` in `src/lib/content/citations.ts`. `CitationList` accepts `registryId` (module `citationIds`) or explicit `citationIds`; renders MLA in linked `<ol>` with `rel="noopener noreferrer"`.
- `biome.json` ignores `.next`, `node_modules`, and `storybook-static` so local builds do not fail lint.
- `DerivedRelatedDocs` derives Phase 1 `same-variant-group` via `deriveRelatedDocGroups` in `src/lib/content/related-docs.ts`; peer modules need registry JSON with matching `variantGroup`. Module links use `modulePageHref(slug)` → `/docs/modules/<slug>`.
