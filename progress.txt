## Codebase Patterns
- App Router tracing boundary: route modules import thin facades (`glossary.ts`, `tags.ts`, `ui-messages.ts`, `module-page.ts`, `glossary-page.ts`) that use `await import("./…-load")` or `await import("./pages")` inside async loaders; filesystem scanners live in `*-load.ts`, `pages.ts`, `registry.ts`, `tag-messages.ts`.
- Pure registry lookups without fs: `registry-index.ts` exports `RegistryIndexes`, `RegistryRecord`, and `getRegistryRecord`; discovery facades import these instead of `registry.ts`.
- Content filesystem roots: `src/lib/content/content-paths.ts` exports `CONTENT_ROOT`, `DOCS_ROOT`, `GLOSSARY_DOCS_ROOT`, `MODULES_DOCS_ROOT`, `REGISTRY_ROOT`, `MESSAGES_ROOT`, `TAG_MESSAGES_ROOT`, and Phase 1 fixture dirs; loaders should import these instead of `join(process.cwd(), "src/content/...")`.
- Turbopack NFT tracing regression: `buildOutputHasTurbopackWholeProjectTracingWarning` in `src/lib/build/turbopack-nft-tracing-warning.ts` guards known diagnostic fragments; `src/tests/build/next-build-tracing-warning.test.ts` runs a clean `bun run build` (`.next` removed first) and fails on combined stdout/stderr matches.
- Registry validation negative tests: use temp `registryRoot` + empty `docsRoot` fixtures in `validate-registry.test.ts`; assert `unresolved-reference` for broken `relatedIds`/`citationIds` (message includes `${record.id}`, field name, and missing id) and `unresolved-tag` for unknown tag slugs; baseline passes via `validateRegistryContent()` with no options and `make validate-data`.
- Fumadocs codegen: `pretypecheck` and `pretest` in `package.json` both run `fumadocs-mdx` so `make typecheck` and `make test` work on fresh clones without a gitignored `.source/` directory; `src/lib/source.ts` imports `../../.source/server`.
- Home discovery links: `/` must link to `/search`, `/docs/architecture`, `/docs/glossary`, `/docs/glossary/token`, `/tags`, and `/docs/modules/grouped-query-attention` via `HomeBrowseLink` or explicit `Link`; search section uses `HomeSearchEntry` plus a `/search` text link from `home.searchPageLinkTitle`.
- Primary nav: `getPrimaryNavItems` in `src/components/layout/primary-nav.ts` is the single source for DocsShell links (`/`, `/search`, `/docs/architecture`, `/docs/glossary`, `/tags`); use `PRIMARY_NAV_LINK_CLASS` for ring-token focus styles.
- Architecture index at `/docs/architecture`: `ArchitectureIndexList` / `ArchitectureEmptyState` mirror glossary index; `loadPublishedArchitectureEntries` includes `docs/architecture/*` paths and pages whose registry concept has `conceptType: architecture` (Phase 1 lists token at `/docs/glossary/token` via `concept.token`).
- Phase 1 search assertions: use `resultsIncludeSampleModule` in `src/tests/search/helpers.ts` because Fumadocs advanced search returns multiple fragments per URL and `attention` may rank glossary hits above the GQA page row.
- Search documents are built from `loadPublishedDocsPages(locale)` + `loadRegistry()` via `buildSearchDocuments` / `buildSearchDocumentsForLocale` in `src/lib/search/build-documents.ts`; body text and headings come from colocated `messages/<locale>.json`, not MDX prose.
- Registry aliases and tag record aliases merge into `SearchDocument.aliases` through `tagSearchTerms()`; page and registry `tags` populate `SearchDocument.tags` and facets.
- Static Orama snapshot: run `make build-search-index` (writes gitignored `src/generated/search-index.json`); `exportOramaIndexSnapshot` returns `OramaSnapshotDocument` entries with top-level `kind` and `tags` alongside Fumadocs structured data; Orama DB records mirror those fields via `toOramaRecord`.
- Fumadocs advanced search uses `toAdvancedSearchIndexes` in `search-server.ts` at runtime without the generated artifact.
- Published docs pages need `page.mdx` frontmatter (`kind`, `registryId`, `status: published`) plus colocated messages; registry JSON under `src/content/registry/<kind>/`.
- Phase 1 sample content: GQA module at `src/content/docs/modules/grouped-query-attention`, token glossary at `src/content/docs/glossary/token`.
- Phase 1 static route verification: `src/lib/build/verify-phase-1-static-routes.ts` exports `PHASE_1_STATIC_ROUTES`, `missingPhase1StaticRoutes`, and `verifyPhase1StaticRoutesFromManifest`; `scripts/verify-phase-1-static-routes.ts` accepts an optional manifest path argv for fixture tests; `make build` runs the script against `.next/app-path-routes-manifest.json`.
- GitHub Actions CI: `.github/workflows/ci.yml` runs on `pull_request` and pushes to `main`; `oven-sh/setup-bun@v2`, `bun install --frozen-lockfile`, then `make ci` (no secrets).
- Search API tests: `src/tests/search/search-api.test.ts` covers `docsSearchApi.search` ranking for GQA/title/attention/KV-cache variants and Token/tokens/tokenizer; use `resultsIncludeUrl`, `resultsIncludeSampleModule`, and `resultsIncludeTokenGlossary` from `helpers.ts`.
- UI search client tests: `src/tests/search/search-client.test.ts` mocks `docsSearchApi.staticGET()` via `globalThis.fetch` and asserts `createDocsSearchClient` ranking; mirror token/GQA coverage from API tests in `search-result-meta.test.ts` for `resolveSearchResultMeta` / `getMatchedTags` (token glossary meta uses `attention` tag, not a `token` slug).
- Global search UI: `SearchDialog`/`SearchTrigger` register via `AppProviders` → `RootProvider` in `src/app/layout.tsx`; `DocsShell` renders `SearchTrigger` in the top nav for every `(site)` route; `HomeSearchEntry` opens the same dialog from `/`.

- Turbopack NFT containment: when stories 002–004 eliminate the whole-project tracing warning, keep only `turbopack.root` in `next.config.ts`; do not add `outputFileTracing*` containment or a `progress.txt` residual-warning entry.
- Glossary listing: `loadPublishedGlossaryEntries` and `listPublishedGlossaryPages` both use `loadPublishedDocsPages` via dynamic import in `glossary.ts`; the duplicate `glossary-pages.ts` scanner was removed.
- Glossary index at `/docs/glossary` uses `loadPublishedGlossaryEntries` (filters `kind: glossary`); empty state mirrors architecture index with home link + `SearchTrigger`.
- Tags index at `/tags` uses `loadPublishedTagIndexGroups` from published `tag` registry records; localized title/summary live in `src/content/registry/tags/messages/<slug>.<locale>.json`; `loadRegistry` skips `messages` subdirectories.
- Tag index groups by `category` (schema order via `TAG_CATEGORY_ORDER`); tag slug chips use monochrome `border-border` / `text-muted-foreground` styling.
- Tag landing at `/tags/[slug]` uses `loadTagResourceGroups` in `src/lib/content/tag-resources.ts` (matches tag on MDX frontmatter and registry record tags); `TagResourceList` groups by page kind and omits empty groups; `TagSearchHandoff` + `search-prefill.ts` prefill the global search dialog query when opened from a tag page; add a `/search?tag=<slug>` link via `tagLanding.searchEntryLink` for documented search entry handoffs per architectural checklist.
- Search entry at `/search` uses `searchEntry.*` message keys, documents canonical `/search` and optional `?tag=<slug>` handoffs, and `SearchEntryHandoff` (client, Suspense-wrapped) opens the global dialog with optional tag prefill via `setPendingSearchQuery`.
- Glossary index page render tests mirror architecture index: assert Token card and `href="/docs/glossary/token"` in `glossary-index.test.ts`; discovery smoke uses `alsoExpectInHtml: "Token"` for `/docs/glossary`.
