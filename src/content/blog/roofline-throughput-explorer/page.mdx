## Why active weight reads can cap throughput

Serving a language model repeatedly reads weights from memory. When the memory subsystem cannot supply bytes fast enough, compute sits idle and throughput stops growing even if the accelerator has spare FLOPs.

The roofline view plots achievable decode throughput (tokens per second) against memory bandwidth. The sloped boundary is the memory-bound limit: larger active weight reads and wider parameter encodings lower achievable throughput because each forward pass must move more bytes.
